Casting Karma LLC · The Whole System

One master project. Many chats. A portfolio of micro-companies.

Casting Karma is run inside Claude Cowork as a single master project. Claude operates each business function as its own chat ("lane"), acting as CEO / CTO / CMO. The chats don't share memory — so the living documents saved on the hard drive are the shared brain. Read the docs, do the work, write it back. That loop is what turns one operator into a whole company.

1 The brain — where Claude & Jamin stand

🧠 Claude (Cowork) — the operator

Runs every lane end-to-end: researches, builds, ships, and grows the businesses with high autonomy. One mind, many chats — each chat is a focused role.

🧑‍✈️ Jamin — strategic partner & final approver

Sets direction and approves. Spending money, creating accounts, entering credentials, publishing, DNS, anything irreversible = Jamin. Claude pauses at these "human walls."

🔗 No shared memory

Two chats can't see each other's context. The living docs on disk are the only shared truth — which is why writing changes back is the whole job.

2 Shared memory — the living docs (single source of truth)

Read the relevant ones before acting; write every change back after. If it isn't written to a living doc, it doesn't exist for the other chats.

  • ★ Activity LogOne chronological feed of every update, all lanes. Read first.
  • NOW — State30-second snapshot of every lane + open handoffs.
  • Operating MapThe rulebook: lanes, rules of the road, chat registry.
  • Portfolio TrackerPer-project status, stage, MRR, key links.
  • Vault + RenewalsEvery account, which email, where each secret lives, cost & renewal.
  • StandardsStack · Build (Definition of Done) · Email · Connectors register.
  • Per appRunning Log · Feature Matrix · Human-Walls Queue · Notification Map.
Where they live — on the hard drive

Casting Karma LLC/ ├─ 00_Operations/ maps, standards, registers, logs ├─ 01_Vault/ accounts + where secrets live ├─ 02_Research/ Discovery's pipeline ├─ 03_Businesses/ one folder per app + the dashboard ├─ 04_Distribution/GTM playbooks + marketing ├─ 05_Finance/ ledger + P&L (CK-Finance) ├─ 06_Legal/ legal register + packets (CK-Legal) └─ 99_Archive/ superseded, kept for trail

Secrets themselves never sit here — real keys live in Apple Passwords; the Vault only indexes what exists and where.
READ ▸ NOW → Activity Log → Operating Map → the docs for what you're touching.
WRITE ▾ your app's Running Log (newest on top) + one line to the Activity Log. That line is how every other lane sees it.
Single writer — Operations alone syncs the Tracker + dashboard from the Activity Log, so nothing forks.
